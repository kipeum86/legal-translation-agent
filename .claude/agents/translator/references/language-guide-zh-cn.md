# Simplified Chinese / PRC (简体中文) Legal Translation Guide

## Register & Formality

- **Statutes**: Third person, impersonal. No 你/您 — uses abstract subjects (公民, 当事人, 个人信息处理者)
- **Contracts**: 甲方/乙方 (Party A/Party B). Formal register throughout.
- **Consumer-facing docs** (ToS, Privacy Policy): **您** (formal "you"). Never use informal 你.

### Key Obligation Markers
| Chinese | English | Usage |
|---|---|---|
| **应当** | shall (mandatory) | Mandatory obligation |
| **可以** | may (permissive) | Discretionary |
| **不得** | shall not / may not | Prohibition |
| **必须** | must | Stronger than 应当 |

### Contract Conventions
- **本合同** = this Contract/Agreement
- **甲方** = Party A (service provider, employer, disclosing party)
- **乙方** = Party B (customer, employee, receiving party)
- **双方** = both Parties
- Company **公章** (chop/seal) is primary authentication, often more authoritative than signature

## Article Numbering Convention

### Full Hierarchy (per 立法法 Article 61)
| Level | Chinese | Numbering | Display |
|---|---|---|---|
| 编 (Part) | 第一编 | Chinese numerals | 第一编, 第二编 |
| 章 (Chapter) | 第一章 | Chinese numerals | 第一章 总则 |
| 节 (Section) | 第一节 | Chinese numerals | 第一节 |
| **条 (Article)** | **第一条** | **Chinese numerals, sequential throughout** | **第一条, 第二条...第一千二百六十条** |
| **款 (Paragraph)** | **(NOT numbered)** | **Identified by position only** | Just a new paragraph within 条 |
| **项 (Item)** | **(一)(二)(三)** | **Chinese numerals in parentheses** | (一)、(二)、(三) |
| **目 (Sub-item)** | **1. 2. 3.** | **Arabic numerals** | 1、2、3 |

### Critical Rules
- **款 is NOT numbered** — just separate paragraphs within an article. Cited as 第一款, 第二款 by counting.
- **条** numbered sequentially through entire law (not per chapter)
- **项** and **目** restart numbering within each 款

### Citation Format
```
《民法典》第五百零九条第二款第一项
```

## Defined Term Convention

### In Contracts
```
北京XX科技有限公司（以下简称"甲方"）
"保密信息"是指...
```
- Defined terms in Chinese quotation marks **""** (NOT Western quotes)
- Pattern: **以下简称"abbreviated term"**
- **是指** for "means/refers to"

### In Legislation
```
《中华人民共和国民事诉讼法》（以下简称民事诉讼法）
```
- Abbreviation after 以下简称 does NOT use 《》

## Emphasis Convention

| Method | Context |
|---|---|
| **Bold (加粗)** | Primary emphasis in legal documents |
| **Bold + underline** | Critical clauses (liability, disclaimers) |
| 【】 | Some commercial legal docs for headers (NOT in legislation) |

- **No italics** in Chinese legal documents — they reduce CJK readability
- **No ALL CAPS equivalent** — Chinese has no case distinction

### When Translating FROM English ALL CAPS
Map ALL CAPS → **Bold** (加粗) or **Bold + underline**

## Legal Citation Format

### Statute Names: 《》 (book title marks)
```
《中华人民共和国民法典》
《个人信息保护法》
```
Abbreviated after first use: 《民法典》

### Judicial Interpretations
```
法释〔2009〕14号
```
Note: 〔〕 brackets (NOT regular parentheses) for year

### Court Decisions
```
(2020)最高法民申1094号
```
Format: (Year)CourtAbbrev+CaseType+Number号

## PRC vs Taiwan: MUST NEVER MIX

**Never insert Traditional characters into Simplified text.** Key differences:

| Concept | PRC (简体) | Taiwan (正體) |
|---|---|---|
| Software | 软件 | 軟體 |
| Information | 信息 | 資訊 |
| Network | 网络 | 網路 |
| Server | 服务器 | 伺服器 |
| Contract | 合同 | 合約/契約 |
| Program | 程序 | 程式 |
| Digital | 数字/数码 | 數位 |
| File | 文件 | 檔案 |
| Data | 数据 | 資料 |
| IP rights | 知识产权 | 智慧財產權 |

## Common Pitfalls

1. **个人信息处理者 ≠ data processor** — It's closer to GDPR "data controller"
2. **Terminological inconsistency**: Using 终止 and 解除 interchangeably (they are legally distinct in PRC law)
3. **Character contamination**: Accidentally using Traditional characters
4. **Register mismatch**: Colloquial expressions in formal legal text
5. **Mixing PRC and common-law concepts**: PRC is a civil law jurisdiction

## Common Document Structures

### 隐私政策 (Privacy Policy) — per PIPL
1. 引言 → 2. 如何收集和使用个人信息 → 3. Cookie等技术 → 4. 共享、转让、公开披露 → 5. 保护措施 → 6. 您的权利 → 7. 未成年人信息 → 8. 跨境传输 → 9. 更新 → 10. 联系方式

### 保密协议 (NDA)
1. 协议双方 → 2. 保密信息定义 → 3. 保密义务 → 4. 使用限制 → 5. 例外 → 6. 保密期限 → 7. 违约责任 → 8. 争议解决 → 9. 适用法律 → 10. 其他
