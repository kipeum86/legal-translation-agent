# Japanese (日本語) Legal Translation Guide

## Register & Formality

| Document Type | Register | Notes |
|---|---|---|
| **法律条文** (Statutes) | **である体** | Always. No exceptions. |
| **契約書** (Contracts) | **である体** (dominant) | Traditional; still majority practice |
| **利用規約** (Terms of Service) | **です/ます体** or **である体** | です/ます increasingly common for consumer-facing |
| **プライバシーポリシー** (Privacy Policy) | **です/ます体** | Consumer-facing, polite form standard |
| **訴状** (Court pleadings) | **である体** | |

**Critical rule**: NEVER mix registers within a single document.

### Key Obligation Markers
| Japanese | English | Usage |
|---|---|---|
| **しなければならない** | must / shall | Mandatory obligation (strongest) |
| **するものとする** | shall / is to | Softer mandatory (common in statutes) |
| **することができる** | may | Permission/authority (NOT ability) |
| **してはならない** | shall not / must not | Prohibition |

## Article Numbering Convention

### Full Hierarchy (per 参議院法制局)
| Level | Japanese | Numbering | Display |
|---|---|---|---|
| 編 (Part) | 第一編 | Chinese numerals | 第一編, 第二編 |
| 章 (Chapter) | 第一章 | Chinese numerals | 第一章, 第二章 |
| 節 (Section) | 第一節 | Chinese numerals | |
| 款 (Subsection) | 第一款 | Chinese numerals | |
| 目 (Division) | 第一目 | Chinese numerals | |
| **条 (Article)** | **第一条** | **Chinese numerals (漢数字) in statutes** | 第一条, 第二条 |
| **項 (Paragraph)** | **2, 3, 4...** | **Arabic numerals; paragraph 1 has NO number** | ２ / ３ / ４ |
| **号 (Item)** | **一、二、三** | **Chinese numerals** | 一, 二, 三 |
| Sub-item 1 | **イ、ロ、ハ** | **Katakana (iroha order)** | イ, ロ, ハ, ニ, ホ, ヘ, ト... |
| Sub-item 2 | **(1)(2)(3)** | Arabic in parentheses | |
| Sub-item 3 | **(i)(ii)(iii)** | Lowercase roman | |

### Critical Rules
- **Paragraph 1 (第1項) has NO visible number.** Numbering starts from paragraph 2 (２, ３, ４...)
- **号 uses Chinese numerals**: 一、二、三 (NOT 1. 2. 3.)
- **Sub-items use katakana in iroha order**: イ、ロ、ハ、ニ、ホ (NOT ア、イ、ウ、エ、オ)
  - Some newer laws use ア、イ、ウ order — check consistency within the document

### Citation Format
```
第八条第二項第一号イ
```

## Defined Term Convention

### Standard Pattern (exact punctuation)
```
[Full term]（以下「[abbreviated term]」という。）
```
- Full-width parentheses （）
- **「」** (corner brackets) for defined term
- **という。** ends with period INSIDE parenthesis

### Real Example (民法 Art. 522)
```
契約の内容を示してその締結を申し入れる意思表示（以下「申込み」という。）
```

### Scope Limitation Variants
| Pattern | Scope |
|---|---|
| 以下「X」という。 | Entire law |
| 以下この条において「X」という。 | This Article only |
| 以下この章において「X」という。 | This Chapter only |

### Bracket Rules
| Bracket | Usage |
|---|---|
| **「」** (かぎ括弧) | Defined terms, quotations — **standard** |
| **『』** (二重かぎ) | Quote within quote; publication titles |
| **【】** (隅付き括弧) | Section headers in contracts/business docs (NOT in statutes) |

## Emphasis Convention

Japanese has NO case distinction and NO native italic tradition.

| Method | Usage |
|---|---|
| **【】** | Section headers in contracts/ToS: 【第1条（目的）】 |
| **Bold (太字)** | Emphasis in modern/digital documents |
| **Underline (下線)** | Traditional emphasis (less common in print) |
| **傍点 (Side dots)** | Traditional typographic emphasis (dots beside characters) |
| **「」** | Highlighting specific terms |

### Critical Rules
- **Statutes NEVER use bold, underline, or 【】** — structure alone provides hierarchy
- **Contracts** commonly use 【】 for headers
- **NEVER apply italics to Japanese text** — it merely slants characters and reduces readability

### When Translating FROM English ALL CAPS
Map ALL CAPS → **Bold** or **Bold + 【】 header**

## Legal Citation Format

### Statutes (法律)
```
民法第709条
個人情報保護法第2条第1項第1号
```

### Law Number (法律番号)
```
[Era]X年法律第X号
```
Examples:
- 民法: 明治二十九年法律第八十九号
- 個人情報保護法: 平成十五年法律第五十七号

### Court Decisions
```
最判平成29年1月31日民集71巻1号48頁
```
Format: [Court abbrev][Era year]年[date][Reporter][Vol]巻[No]号[Page]頁

### Reporter Abbreviations
| Abbreviation | Full Name |
|---|---|
| 民集 | 最高裁判所民事判例集 |
| 刑集 | 最高裁判所刑事判例集 |
| 判時 | 判例時報 |
| 判タ | 判例タイムズ |

## Date Format

### Two Systems
| System | Format | Example |
|---|---|---|
| **和暦** (Japanese era) | 令和X年X月X日 | 令和8年3月16日 |
| **西暦** (Western) | 20XX年X月X日 | 2026年3月16日 |

### Usage
- **Statutes, courts, government**: 和暦 (always)
- **Domestic contracts**: 和暦 (majority) or 西暦
- **International contracts**: 西暦
- **ToS, Privacy Policy**: Either (西暦 increasingly common)

**Never mix 和暦 and 西暦 in one document.**

### Era Conversion
| Era | Start | Formula |
|---|---|---|
| 令和 | 2019 | 令和X = 2018 + X |
| 平成 | 1989 | 平成X = 1988 + X |
| 昭和 | 1926 | 昭和X = 1925 + X |

## Key Legal Terminology

### APPI (個人情報保護法)
| Japanese | English |
|---|---|
| 個人情報 | Personal information |
| 個人データ | Personal data |
| 保有個人データ | Retained personal data |
| 要配慮個人情報 | Special care-required personal information |
| 個人情報取扱事業者 | Business operator handling personal information |
| 匿名加工情報 | Anonymously processed information |
| 仮名加工情報 | Pseudonymously processed information |

### Critical Translation Trap: 善意/悪意
- **善意** (ぜんい) = "**without knowledge**" (of a fact) — NOT "good faith"
- **悪意** (あくい) = "**with knowledge**" (of a fact) — NOT "bad faith"
- **信義誠実の原則** (信義則) = the actual "good faith" principle

### 又は/若しくは and 及び/並びに
| Connector | Level | Example |
|---|---|---|
| **又は** | Higher "or" | A若しくはB**又は**C = (A or B) or C |
| **若しくは** | Lower "or" | A**若しくは**B又はC |
| **並びに** | Higher "and" | A及びB**並びに**C = (A and B) and C |
| **及び** | Lower "and" | A**及び**B並びにC |

### Contract Terms
| Japanese | English |
|---|---|
| 甲・乙・丙 | Party A/B/C |
| 解除 | Cancellation/Rescission |
| 解約 | Termination |
| 秘密保持 | Confidentiality |

## Common Pitfalls: 翻訳調 (Translationese)

| Problem | Example | Better |
|---|---|---|
| Sentence too long | 200-word English → single Japanese sentence | Break into shorter sentences |
| Passive overuse | ~されるものとする | ~するものとする (active voice) |
| Unnatural word order | SVO calqued from English | Natural Japanese SOV |
| Excessive katakana | コンフィデンシャリティ | 秘密保持 (use established terms) |
| Register mixing | です/ます + である in one doc | Choose one, maintain throughout |
| 善意/悪意 confusion | "good faith"/"bad faith" | "without/with knowledge" |

## Common Document Structures

### 利用規約 (Terms of Service)
1. 総則 → 2. 定義 → 3. アカウント登録 → 4. サービス内容 → 5. 料金・支払い → 6. 禁止事項 → 7. 知的財産権 → 8. ユーザーコンテンツ → 9. 個人情報 → 10. 免責事項 → 11. サービス変更・終了 → 12. 損害賠償 → 13. 規約変更 → 14. 準拠法・管轄

### プライバシーポリシー (Privacy Policy)
1. 個人情報の定義 → 2. 収集方法 → 3. 利用目的 → 4. 第三者提供 → 5. 共同利用 → 6. 安全管理措置 → 7. Cookie → 8. 開示・訂正・削除 → 9. 匿名加工情報 → 10. お問い合わせ → 11. 変更
