# Traditional Chinese / Taiwan (正體中文) Legal Translation Guide

## Register & Formality

- **Statutes**: Impersonal constructions. No 你/您.
- **Contracts**: 甲方/乙方. Both **合約** and **契約** are valid (合約 more common in business, 契約 more formal).
- **Consumer-facing docs**: **您** exclusively. Never 你.

### Formal Register Markers (distinct from PRC)
| Taiwan (Prefer) | PRC Equivalent (Avoid) | Meaning |
|---|---|---|
| **係指** | 是指 | means/refers to |
| **之** | 的 (in formal contexts) | possessive/structural particle |
| **應** | 应 | shall (mandatory) |
| **得** | 可以 | may (permissive) |
| **不得** | 不得 | shall not |
| **及/暨** | 和/跟 | and |
| **惟/但** | 但是 | however/proviso |
| **合約/契約** | 合同 | contract |
| **本合約** | 本合同 | this Agreement |

## Article Numbering Convention — DIFFERENT FROM PRC

### Taiwan Hierarchy (per 中央法規標準法 第8條)
| Level | Name | Numbering | Display |
|---|---|---|---|
| **條** (Article) | 第一條 | Chinese numerals | 第一條、第二條 |
| **項** (Paragraph) | **NOT numbered** | By position | (just a new paragraph) |
| **款** (Subparagraph) | **一、二、三** | Chinese numerals + 、 | 一、二、三 |
| **目** (Item) | **(一)(二)(三)** | Parenthesized numerals | （一）、（二）、（三） |
| (sub-item) | **１、２、３** | Arabic numerals | 1、2、3 |

### CRITICAL: 款 and 項 SWAP positions vs PRC

| Level | **Taiwan** | **PRC** |
|---|---|---|
| Level 2 (under 條) | **項** (paragraph, unnumbered) | **款** (paragraph, unnumbered) |
| Level 3 | **款** (一、二、三) | **项** ((一)(二)(三)) |

This is the single most important structural difference between Taiwan and PRC legal documents.

## Defined Term Convention

### Standard Pattern
```
個人資料保護法（以下簡稱「個資法」）
```
Pattern: `Full Name（以下簡稱「Abbreviation」）`

### In Definitions Articles
```
一、「機密資訊」：係指一方向他方揭露之任何非公開資訊，包括但不限於…
二、「揭露方」：係指揭露機密資訊之一方。
```
- **「」** wraps defined terms (NOT "" like PRC)
- **係指** (NOT 是指) for "means/refers to"
- Items numbered with 一、二、三 (款 numbering)

## Emphasis Convention

| Method | Usage |
|---|---|
| **「」** | Defined terms, statute names, abbreviations |
| **Bold** | Clause headings in contracts |
| **『』** | Quotes within quotes (nested) |

### Statute Name Brackets — DIFFERENT FROM PRC
| Context | Taiwan | PRC |
|---|---|---|
| Statute names | **「個人資料保護法」** | 《个人信息保护法》 |
| Book titles | 《》 | 《》 |

**Rule**: Taiwan uses 「」 for statute names. PRC uses 《》. This is a highly visible register marker. Using 《》 for statute names in Taiwan text is incorrect.

## Legal Citation Format

### Statutes
```
依「個人資料保護法」第6條
```

### Court Decisions
```
最高法院94年度台上字第6號民事判決
臺灣高等法院109年度上字第1234號民事判決
```
Format: [法院名] [民國年度]年度[字別]字第[號]號[裁判類型]

**ROC Calendar**: 民國年 = 西元年 - 1911 (2026 = 民國115年)

**Court names use 臺 (NOT 台)**: 臺灣高等法院, 臺灣臺北地方法院

## MUST NEVER MIX: Simplified Characters

### One-to-Many Mapping Traps (auto-conversion errors)
| Simplified | Correct Traditional (Legal) |
|---|---|
| 里 | **裡** (inside) vs 里 (neighborhood) |
| 后 | **後** (after) vs 后 (empress) |
| 发 | **發** (emit/issue) vs 髮 (hair) |
| 干 | **乾** (dry) / **幹** (do) / 干 (interfere) |
| 复 | **複** (duplicate) vs **復** (restore) |
| 系 | **係** (is) vs 系 (system) vs 繫 (tie) |
| 征 | **徵** (levy) vs 征 (expedition) |
| 余 | **餘** (remainder) vs 余 (surname) |

### Common Legal Characters
| Wrong (Simplified) | Correct (Traditional) | Context |
|---|---|---|
| 与 | **與** | 與第三人 |
| 为 | **為** | 為…之目的 |
| 于 | **於** | 適用於… |
| 权 | **權** | 智慧財產權 |
| 责 | **責** | 責任 |
| 违 | **違** | 違約 |
| 应 | **應** | 應遵守 |

### 臺 vs 台 Rule
- **Court names**: Always **臺** (臺灣高等法院)
- **Government documents**: Prefer 臺
- **Commercial text**: 台 is acceptable
- Be consistent within one document
